# GrooBeat ヒアリングシート

お客様と対面で使う、選択式のヒアリングページ。15問に答えると「デザインコンセプト文」が自動生成され、コピー／共有できる。生成された文章はそのままデザイン発注書の頭に貼れる。

- **公開**：GitHub Pages（静的HTML・外部依存ゼロ・サーバー通信なし）
- **完全クライアント完結**：回答の送信・保存・アカウントは一切なし。素材の送付ボタンは `mailto:`（メールアプリを開くだけ）
- **noindex**：社内向けツールのため検索インデックス対象外（`<meta name="robots" content="noindex">`）

---

## ファイル構成

| ファイル | 役割 | 差し替え頻度 |
|---|---|---|
| `index.html` | 共通エンジン（画面遷移・プログレス・コピー／共有・素材コーナー）＋CSS。**種目を知らない** | 低（エンジン改良時のみ） |
| `questions.app.js` | 種目「アプリ制作」の質問セット＋コンセプト生成関数 `buildConcept` | 種目ごとに1本 |
| `config.js` | 素材送付先（メール宛先・件名・チェックリスト）などの設定値 | 宛先変更・LINE移行時 |

**設計思想＝共通エンジン＋質問セット差し替え方式。** 質問セット（`questions.app.js`）を差し替えるだけで、HP版・デザイン版・印刷物版など別種目のシートになる。

---

## 横展開：新しい種目シートの作り方

1. `questions.app.js` をコピーして `questions.hp.js`（例：ホームページ版）を作る。
2. 中の `QUESTIONS` 配列を、その種目の質問に書き換える（下の「質問データの形」参照）。
3. `buildConcept(answers)` を、その種目の文体に合わせて書き換える（コンセプト文の組み立てはここに閉じている）。
4. `window.HEARING_SET = { meta:{...}, questions: QUESTIONS, buildConcept: buildConcept }` の `meta.label` / `meta.title` を種目名に変える。
5. `index.html` の入口画面で、その種目のカードを「準備中」から活性へ切り替える（`index.html` 内 `renderIntro()` の準備中リスト参照）。

> 現状の入口は「アプリ」のみ活性、他（ホームページ／ロゴ・デザイン／看板・印刷物）は「準備中」表示。エンジンが完成しているので、質問セットが揃った種目から順に活性化する。

### 質問データの形（`QUESTIONS` の1要素）

```js
{
  id: 'Q1',                 // 一意なID（answers のキーになる）
  round: 'R1 方向性',        // グルーピング。画面には "R番号" を除いた日本語部分だけ出る
  title: '設問文',
  multi: false,             // true＝複数選択（下部に「次へ」ボタン／単一選択は選んだ瞬間に進む）
  note: '複数選択できます',  // 任意の補足（省略可）
  options: [
    {
      emoji: '🐻',
      label: '選択肢の文言',
      feedback: '選んだ直後に1文で出る「見えてきた像」',
      concept: 'コンセプト文に埋める断片',  // null＝おまかせ等でコンセプトに載せない
      quote: true,          // 任意：付けると「※別途お見積り」バッジが出る
      none: true            // 任意：「特にない／おまかせ」枠（複数選択で他と排他になる）
    }
  ]
}
```

- `feedback`：単一選択で選んだ瞬間に 1.5秒（またはタップで即）表示される。
- `concept`：`buildConcept` が組み立てる断片。**`none: true` の枠は `concept: null`** にして、コンセプト文に載せない。
- `multi: true` の設問では「特にない／おまかせ」（`none: true`）を選ぶと他の選択が外れる（排他）。

### コンセプト文の設計メモ（手戻り防止の要）

- **NG回答（使いたくない色・見られたくない印象）は必ず「避けたいこと」としてコンセプト文に明記**する。嫌いの後出しが手戻りの最大要因なので先に潰す。
- **「やらないこと」は必ず末尾に明記**する（スコープ確定の証拠）。
- おまかせ・特になし・全部前向き検討などの選択でも文章が破綻しないよう、`buildConcept` 側でフォロー文を用意している。

---

## 設定値（`config.js`）

- `materialSend.to` / `subject` / `bodyLines`：素材送付メールの宛先・件名・本文テンプレ。
- 将来 **GrooBeat公式LINE** ができたら、素材の回収導線をここで差し替える（`index.html` の素材コーナーのボタン生成箇所）。
- **`config.js` には顧客名・個人情報・APIキー・秘密情報を一切書かない**（公開リポジトリに載るため）。

## 公開（GitHub Pages）

- リポジトリ：`bunchan-groobeat/groobeat-hearing`
- Settings → Pages → Branch: `main` / root を選択して公開。
- `.nojekyll` を置いてあるので、Jekyll 処理を挟まずそのまま配信される。

---

## やらないこと（本ツールのスコープ外）

- 回答のサーバー送信・保存（対面専用・完全クライアント完結）
- AI動的生成によるコンセプト文（テンプレ結合で生成。将来の検討課題）
- 他種目の質問セット（エンジン完成後に順次追加）
- Firebase連携・アカウント機能
